<Heading level="1">
    FP Module Overview
</Heading>

<Paragraph>
    The <Mark bold>lodash/fp</Mark> module provides an immutable, auto-curried, iteratee-first, data-last version of Lodash. It's designed for functional programming patterns.
</Paragraph>

<Heading level="2">
    Immutability
</Heading>

<Paragraph>
    FP methods never mutate the original data. Instead, they always return new data structures:
</Paragraph>

<BulletedList>
    <Mark>fp.set</Mark>, <Mark>fp.update</Mark>, <Mark>fp.unset</Mark> - return new objects
</BulletedList>
<BulletedList>
    <Mark>fp.pull</Mark>, <Mark>fp.remove</Mark> - return new arrays
</BulletedList>
<BulletedList>
    <Mark>fp.assign</Mark>, <Mark>fp.merge</Mark> - return new objects
</BulletedList>

<Heading level="2">
    Conversion Options
</Heading>

<Paragraph>
    You can convert Lodash methods to FP-style with custom options using <Mark>method.convert()</Mark>:
</Paragraph>

<Paragraph>
    Available options:
</Paragraph>

<BulletedList>
    <Mark>cap</Mark> - Cap iteratee arguments (default: true)
</BulletedList>
<BulletedList>
    <Mark>curry</Mark> - Enable/disable currying (default: true)
</BulletedList>
<BulletedList>
    <Mark>fixed</Mark> - Fix arity (default: true)
</BulletedList>
<BulletedList>
    <Mark>immutable</Mark> - Enable/disable immutability (default: true)
</BulletedList>
<BulletedList>
    <Mark>rearg</Mark> - Enable/disable argument reordering (default: true)
</BulletedList>

<Callout icon="⚛️" blockColor="purple" borderColor="purple">

</Callout>
