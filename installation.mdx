<Heading level="2">
    Module Formats
</Heading>

<Paragraph>
    Lodash supports multiple module formats to fit your build system:
</Paragraph>

<BulletedList>
    <Mark bold>CommonJS:</Mark> <Mark>require('lodash')</Mark> - Node.js style
</BulletedList>
<BulletedList>
    <Mark bold>ES Modules:</Mark> <Mark>import _ from 'lodash-es'</Mark> - Modern JavaScript
</BulletedList>
<BulletedList>
    <Mark bold>UMD:</Mark> Works in browser, Node.js, and AMD
</BulletedList>
<BulletedList>
    <Mark bold>AMD:</Mark> <Mark>require(['lodash-amd'], function(_) { ... })</Mark>
</BulletedList>

<Heading level="2">
    Build Tools Integration
</Heading>

<Paragraph>
    Lodash works well with popular build tools:
</Paragraph>

<BulletedList>
    <Mark bold>Webpack:</Mark> Use <Mark>lodash-webpack-plugin</Mark> for smaller bundles
</BulletedList>
<BulletedList>
    <Mark bold>Rollup:</Mark> Cherry-pick methods for tree-shaking
</BulletedList>
<BulletedList>
    <Mark bold>Babel:</Mark> Use <Mark>babel-plugin-lodash</Mark> to automate imports
</BulletedList>

<Callout icon="✅" blockColor="green" borderColor="green">
    For modern projects, consider using the ES modules build (<Mark>lodash-es</Mark>) for better tree-shaking support.
</Callout>
