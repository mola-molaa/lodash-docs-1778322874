<Heading level="1">
    Architecture Overview
</Heading>

<Paragraph>
    Lodash is designed with modularity and performance in mind. This page explains the high-level architecture of the library.
</Paragraph>

<Heading level="2">
    Core Design Principles
</Heading>

<BulletedList>
    <Mark bold>Modularity:</Mark> Each method is available as a separate module
</BulletedList>
<BulletedList>
    <Mark bold>Performance:</Mark> Optimized for modern JavaScript engines
</BulletedList>
<BulletedList>
    <Mark bold>Consistency:</Mark> Predictable behavior across environments
</BulletedList>
<BulletedList>
    <Mark bold>Flexibility:</Mark> Multiple build formats and module systems
</BulletedList>

<Heading level="2">
    Build System
</Heading>

<Paragraph>
    Lodash uses a custom build system to generate different output formats:
</Paragraph>

<BulletedList>
    Main build system in <Mark>lib/main/</Mark> - handles core Lodash builds
</BulletedList>
<BulletedList>
    FP build system in <Mark>lib/fp/</Mark> - handles functional programming builds
</BulletedList>
<BulletedList>
    Common utilities in <Mark>lib/common/</Mark> - shared build tools
</BulletedList>

<Heading level="2">
    FP Module Architecture
</Heading>

<Paragraph>
    The FP module provides an immutable, auto-curried, iteratee-first, data-last version of Lodash:
</Paragraph>

<BulletedList>
    <Mark bold>Immutable:</Mark> Methods don't mutate original data
</BulletedList>
<BulletedList>
    <Mark bold>Auto-curried:</Mark> Functions can be partially applied
</BulletedList>
<BulletedList>
    <Mark bold>Argument reordering:</Mark> Iteratee-first, data-last pattern
</BulletedList>
<BulletedList>
    <Mark bold>Placeholder support:</Mark> Use <Mark>fp.__</Mark> for argument placeholders
</BulletedList>
